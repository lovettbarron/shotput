Purpose: Enables capturing screenshots of pages behind complex auth (OAuth, 2FA, CAPTCHAs) that cannot be handled by simple cookie injection.
Output: `interactiveLogin()` in auth.ts, `shotput_login` MCP tool, human verification of headed browser flow.

  <action>
    1. Add `interactiveLogin` method to `SessionManager` in `src/auth.ts`:
       - Signature: `async interactiveLogin(url: string, sessionName: string): Promise<StorageState>`
       - Guard against concurrent login: add a private `loginInProgress: boolean` field, initialized to false
       - If `loginInProgress` is true, throw `Error("Another interactive login is already in progress. Complete or close the current browser window first.")`
       - Set `loginInProgress = true` in a try/finally block
       - Import `chromium` from `playwright` at the top of the file
       - Launch headed browser: `const browser = await chromium.launch({ headless: false })`
       - Create context: `const context = await browser.newContext()`
       - Create page: `const page = await context.newPage()`
       - Navigate: `await page.goto(url)`
       - Wait for user to finish: use `await new Promise<void>((resolve) => { browser.on('disconnected', () => resolve()); })` -- this resolves when user closes the browser window
       - After browser closes, capture state BEFORE close: Actually, we need to capture state before browser disconnects. Use a different approach:
         - `let state: StorageState | null = null;`
         - Set up an interval that captures `context.storageState()` every 2 seconds into `state` variable (silent catch on errors)
         - Wait for browser disconnect event
         - Clear the interval
         - If `state` is null, throw Error("No session state captured. The browser may have closed before login completed.")
       - Store session: `this.storeSession(sessionName, state)`
       - Set `loginInProgress = false` in finally block
       - Return the stored state
       - CRITICAL (AUTH-04): Do not log any cookie values. Only log status messages like "Interactive login started" or "Session captured".

    2. Add test in `tests/auth.test.ts`:
       - `describe("interactive login", () => { ... })`
       - Test: "rejects concurrent login attempts" -- call interactiveLogin but mock chromium.launch to return a browser that never disconnects; while that's pending, call interactiveLogin again and expect it to throw "Another interactive login is already in progress"
       - Use `vi.mock('playwright', ...)` to mock chromium.launch for this test group
       - Test: "stores session after browser close" -- mock chromium to simulate: launch returns mock browser, context.storageState() returns a fake StorageState, browser 'disconnected' event fires after short delay. Verify session is stored.

    3. Register `shotput_login` tool in `src/server.ts`:
       - Name: `shotput_login`
       - Description: "Open a visible browser window for manual login. Log in to the website, then close the browser window when done. The session will be saved and can be used with shotput_capture's sessionName parameter. Requires a display (won't work on headless servers)."
       - Parameters:
         - `url`: z.string().url().describe("URL of the login page to open")
         - `sessionName`: z.string().default("default").describe("Label for this session (use with shotput_capture's sessionName)")
       - Handler:
         - try/catch around `getSessionManager().interactiveLogin(url, sessionName)`
         - On success: return `{ content: [{ type: "text", text: "Login session '${sessionName}' captured successfully. Use sessionName: '${sessionName}' in shotput_capture to access authenticated pages." }] }`
         - On error: if message includes "display" or "XServer", return helpful error about needing a display. Otherwise rethrow.

    4. Register `shotput_list_sessions` tool in `src/server.ts`:
       - Name: `shotput_list_sessions`
       - Description: "List all stored authentication sessions."
       - Parameters: none (empty object `{}`)
       - Handler: calls `getSessionManager().listSessions()`, returns the list as text. If empty, returns "No active sessions."
