    - session retrieval: after storing a session with label "mysite", getSession("mysite") returns the StorageState object
    - session persistence: storing session "s1", then calling getSession("s1") multiple times returns the same object each time
    - unknown session: getSession("nonexistent") returns undefined
    - clear session: after clearSession("s1"), getSession("s1") returns undefined
    - clear all: after storing multiple sessions, clearAllSessions() leaves the store empty
    - no credentials logging: session manager module does not call console.log, console.warn, or console.error with cookie values (structural check -- ensure no logging of sensitive data)
    - cookie defaults: cookies without optional fields get sensible defaults (path: "/", httpOnly: false, secure: false, sameSite: "None", expires: -1)
  </behavior>
  <action>
    1. Add to `src/types.ts`:
       - `sessionName?: string` field to `CaptureParams` interface

       - `CookieInput` interface for the MCP tool input (subset of CookieParam with optional fields for path, httpOnly, secure, sameSite, expires)

       - Private `sessions: Map<string, StorageState>` for in-memory storage
       - `storeSession(label: string, state: StorageState): void`

       - `clearSession(label: string): void`
       - `clearAllSessions(): void`
       - `listSessions(): string[]`
       - `createSessionFromCookies(label: string, cookies: CookieInput[]): StorageState` -- converts CookieInput[] to full StorageState with defaults, stores it, and returns it
       - Export `getSessionManager(): SessionManager` singleton accessor
       - CRITICAL (AUTH-04): No logging of cookie values anywhere in this module. No console.log, no debug output with session content.

       - Use `<script>` to check `document.cookie` and update a `<div id="auth-status">`

<task type="auto">
  <name>Task 2: Integrate sessions into capture pipeline and MCP tools</name>
  <files>src/capture.ts, src/server.ts</files>
  <action>
    1. Modify `src/capture.ts` `captureScreenshot()`:
       - After `const browser = await getBrowser();`, before creating context:
       - Import `getSessionManager` from `./auth.js`
       - If `params.sessionName` is provided, call `getSessionManager().getSession(params.sessionName)`
       - If session exists, pass `storageState` to `browser.newContext({ ..., storageState })`
       - If sessionName is provided but session not found, set `warning = "Session '${params.sessionName}' not found, capturing without authentication"`

    2. Modify `src/server.ts`:
       - Add `sessionName` parameter to `shotput_capture` tool schema: `z.string().optional().describe("Name of a stored auth session to use for this capture")`
       - Pass `sessionName` through to `captureParams`

       - Register new `shotput_set_cookies` tool:
         - Name: `shotput_set_cookies`
         - Description: "Inject cookies for authenticated page captures. Stores cookies as a named session that can be referenced by shotput_capture's sessionName parameter."
         - Parameters:
           - `sessionName`: z.string().describe("Label for this session (use with shotput_capture's sessionName)")
           - `cookies`: z.array(z.object({
               name: z.string(),
               value: z.string(),
               domain: z.string(),
               path: z.string().default("/"),
               httpOnly: z.boolean().default(false),
               secure: z.boolean().default(false),
               sameSite: z.enum(["Strict", "Lax", "None"]).default("None"),
               expires: z.number().default(-1),
             })).min(1)
         - Handler: calls `getSessionManager().createSessionFromCookies(sessionName, cookies)`
         - Returns: `{ content: [{ type: "text", text: "Session '${sessionName}' stored with ${cookies.length} cookie(s). Use sessionName: '${sessionName}' in shotput_capture." }] }`

       - Register `shotput_clear_sessions` tool:
         - Name: `shotput_clear_sessions`
         - Description: "Clear stored authentication sessions. Clears a specific session by name, or all sessions if no name provided."
         - Parameters:
           - `sessionName`: z.string().optional()
         - Handler: if sessionName provided, clearSession(sessionName), else clearAllSessions()
         - Returns confirmation text

    3. Ensure `closeBrowser` in browser.ts cleanup also calls `getSessionManager().clearAllSessions()` -- import and call in the `closeBrowser` function. This ensures sessions don't leak if the browser is restarted.
  </action>
