1. Implement `src/server.ts`:
   - Export `createServer(): McpServer`
   - Create McpServer with name: "shotput", version: "0.1.0"
   - Register "shotput_capture" tool with description "Capture a screenshot of a web page"
   - Zod schema for params (per CONTEXT.md decisions):
     - url: z.string().url() -- REQUIRED
     - fullPage: z.boolean().default(false)
     - format: z.enum(["png", "jpeg"]).default("png")
     - quality: z.number().min(0).max(100).optional()
     - width: z.number().int().positive().default(1280)
     - height: z.number().int().positive().default(720)
     - scale: z.number().min(1).max(3).default(1)
     - outputDir: z.string().optional()
     - filename: z.string().optional()
     - wait: z.union([z.enum(["networkidle", "domcontentloaded", "load"]), z.number().positive()]).default("networkidle")
     - autoScroll: z.boolean().default(true)
     - timeout: z.number().positive().default(30000)
   - Tool handler:
     - Call captureScreenshot(params) with the validated params
     - Build MCP response content array:
       a. If result.warning: push { type: "text", text: "Warning: {warning}" }
       b. Push { type: "text", text: "Screenshot saved to: {filePath}" }
       c. If result.buffer.length <= 750 * 1024: push { type: "image", data: buffer.toString("base64"), mimeType: "image/{format}" }
       d. Else: push { type: "text", text: "Image too large for inline display. View the saved file." }
     - Return { content }
   - Call registerCleanup() at module level or in createServer()
